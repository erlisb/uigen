# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Initial setup (install deps, generate Prisma client, run migrations)
npm run setup

# Development server (uses Turbopack)
npm run dev

# Build for production
npm run build

# Run tests (Vitest)
npm test

# Run a single test file
npx vitest run src/lib/__tests__/file-system.test.ts

# Lint
npm run lint

# Reset database
npm run db:reset
```

Set `ANTHROPIC_API_KEY` in `.env` to use the real Claude model. Without it, a mock provider returns static demo components.

## Architecture

UIGen is a Next.js 15 app where users chat with Claude AI to generate React components, previewed live in an iframe.

### Data flow

1. User types a message in `ChatInterface` → `ChatContext` (using Vercel AI SDK's `useChat`) sends it to `POST /api/chat/route.ts`
2. The route reconstructs a `VirtualFileSystem` from serialized file data, streams a response from Claude (or `MockLanguageModel`) using tool calls
3. Two tools operate on the VFS: `str_replace_editor` (create/str_replace/insert commands) and `file_manager` (rename/delete)
4. Tool calls stream back to the client; `FileSystemContext.handleToolCall` applies them to the client-side `VirtualFileSystem`
5. `PreviewFrame` watches `refreshTrigger` from `FileSystemContext`, transforms all files with Babel (`@babel/standalone`) via `jsx-transformer.ts`, creates blob URLs, and injects them as an ES module import map into an iframe's `srcdoc`

### Virtual File System

`VirtualFileSystem` (`src/lib/file-system.ts`) is an in-memory tree of `FileNode` objects. It serializes to/from plain objects for API transport and Prisma storage. The root is always `/`. Files generated by AI are never written to disk.

### Preview system

`jsx-transformer.ts` transforms JSX/TSX files using Babel standalone, resolves imports into blob URLs, maps third-party packages to `https://esm.sh/`, and handles the `@/` alias (maps to virtual FS root). The generated HTML uses Tailwind CDN and wraps the entry component in an error boundary.

### Authentication

JWT-based auth via `jose` (server-only, `src/lib/auth.ts`). Sessions stored in `httpOnly` cookies. The middleware (`src/middleware.ts`) protects routes. Anonymous users can work without signing in; their work is tracked in `anon-work-tracker.ts` and can be claimed upon sign-up.

### Persistence

Prisma with SQLite (`prisma/dev.db`). Only authenticated users have projects saved. The `Project` model stores `messages` (JSON array) and `data` (serialized VFS) as JSON strings. Project routes: `/` (new session) and `/[projectId]` (existing project). The database schema is defined in `prisma/schema.prisma` — reference it whenever you need to understand the structure of data stored in the database.

### AI model

`src/lib/provider.ts` returns `claude-haiku-4-5` via `@ai-sdk/anthropic` when `ANTHROPIC_API_KEY` is set, otherwise returns `MockLanguageModel`. The generation system prompt is in `src/lib/prompts/generation.tsx` and enforces: `/App.jsx` as entry point, Tailwind CSS for styling, `@/` import alias for local files.

### Key context providers

Both providers wrap the main layout:
- `FileSystemProvider` — owns the `VirtualFileSystem` instance, exposes file CRUD and `handleToolCall`
- `ChatProvider` (nested inside) — owns `useChat` from Vercel AI SDK, forwards tool calls to `FileSystemProvider`

### Testing

Vitest with jsdom and React Testing Library. Test files live in `__tests__` subdirectories next to the code they test.
